# 手写zustand

代码能力
- zustand 状态管理流行库
- 为什么需要他？
  组件通信 随着层次增加，传递会越来越复杂，所以需要 中央管理共享状态
  管理好状态(财务) 状态共享 
  简单
  非常流行 star数很多 
- 状态管理库有很多家 redux 老牌状态管理库
  复杂
- 读过他的源码 能够手写 
- 核心内容
  - 以store的方式 提供状态
  - 修改状态
  - 响应式更新 订阅发布者模式 
  - 自定义hook
- 手写 实现以下功能
  - 实现 create(高阶函数) 方法 接收一个函数(set,get,store) 创建一个store 
      返回一个状态对象(状态的初始化)，包含状态，修改状态的方法
      返回一个store hook 函数 useXxxStore
  - useXxxStore 可以接收一个函数 按需返回一些状态或方法
    或是不指定参数 直接返回所有状态
  - subscribe 方法
    添加一个订阅者 
    在状态发生改变时执行 响应式 

## 开始实现
- 实现一个最简单的store 
  存储状态 管理状态 怎么存 怎么取 怎么改
- 加入订阅机制 
  抖音的网红(store 发布者 一个) 和 订阅者(订阅关系 多个)
  以后发布者发布内容(状态改变)
  订阅者就会收到通知 并执行自己的 subsribe函数 
  - store 对象上提供一个 subscribe 方法
    接受一个函数
  - state 改变时 ，执行这个函数
