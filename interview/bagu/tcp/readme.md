  FP = TTFB(DNS解析时间(负载均衡+IP查找)) + 建立TCP连接时间/TLS + 
  server执行时间(慢查询) + 响应下载时间 + HTML DOM树构建时间 + CSSOM时间 + 构建渲染树(RenderTree) + 布局树 + 首次渲染

  前端的性能标准 页面加载的速度，页面的打开速度是关键指标
  影响到用户的留存、付费欲望、PV(page view)、UV(user view)

- 一个数据包的旅程
  互联网 实际是一套理念和协议(不同版本)组成的体系架构。
  互联网中的数据是根据数据包来传输的。
  如果数据比较大怎么办？ 利用带宽、多路复用
  将数据拆分成很多小数据包来传输，提升带宽的利用率和并发效率

  二进制数据帧 每个数据包都有自己的头部信息，包含了源地址、目的地址、端口号、校验和等信息。
  数据帧 包含了实际传输的数据。

  - IP地址 将数据包送达目的主机
    Internet Protocol 互联网协议 计算机的地址
    数据包的传输不确定 会丢失 出错

  - 传输层
    负责数据安全到达
    UDP 机器上的某个应用(端口号)
    像是 快递到达了我家，但是需要他的是我家买这个快递的那个人
    UDP ：数据报协议  简单、快速、不保证完整 ，适合 音频视频传输 
    UDP解决不了结构严格的HTML等web 资源

  - TCP
    浏览器请求、邮件 等 要求出具传输可靠性的应用，如果使用UDP 会出现 
    - 数据包丢失 
      过期时间 过期重传
    - 数据包在不同的时间到达接收端  
      序号、组装
    - 速度稍慢，但是完整到达

确认双方接受以及发送数据的能力
- 三次握手
  - 建立连接
  - A 请求 B SEQ j 
  - B 响应 ACK j + 1 以及请求 A SEQ i
  - A 响应 B ACK i + 1
  
  每一方的发送和接受要两次 握手四次
  第二次的时候可以响应对方的发送能力 同时希望对方确认我们的发送能力 
- 四次挥手
  - A 发送 FIN i 表示自己没有数据要发送了
  - B 响应 ACK i + 1 表示确认收到了
  - B 发送 FIN j 表示自己没有数据要发送了
  - A 响应 ACK j + 1 表示确认收到了
