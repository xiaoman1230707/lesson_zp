        本地开发环境映射 (Local Development)
      这是最常见的用途。在开发 Web 应用时，后端服务通常运行在 localhost:8080 或 127.0.0.1:3000。

      痛点：某些功能（如 OAuth 回调、Cookie 域限制、第三方 API 白名单）要求必须使用正式的域名（如 dev.myapp.com），而不能直接用 IP 或 localhost。
      解决方案：
      127.0.0.1   dev.myapp.com
      127.0.0.1   api.dev.myapp.com
      这样，你在浏览器输入 dev.myapp.com，流量就会直接打到你本地的开发服务器上，完美模拟生产环境的域名结构。

- OSI 七层协议
  http 是应用层协议 基于请求和响应的简单协议
  - 物理层 0 和 1 物理介质层
  - 数据链路层 mac 地址 + 数据  上网设备的唯一ID
  - 网络层 两个IP 地址 + mac 地址 + 数据  互联网设备的唯一ID 我的位置和我要去的位置 
  - 传输层 TCP(序号...) + IP 地址 + 数据  
      应用层的服务 规则 确保有效的传输
      数据包有大小的上限
      一个文件会有好多个数据包 分批次 分通道并发传输
      而数据包会丢失 或 乱序 或 重复
      请求重发 TCP/IP IP保证找到，TCP保证数据完整准时到达
      排序
      或者 UDP 数据报协议 视频、音频
      更快 但是 可能会丢失 或 乱序
- 三次握手
  - 会话层 会话ID + 数据  会话管理
  - 表示层 数据编码格式 + 数据  数据转换
  - 应用层 应用层协议 + 数据  应用层服务
