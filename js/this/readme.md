## 如果想拿到time.geekbang.com 怎么办
- this 
  oop的需要 
  在面向对象的方法内 **应该**可以通过this 来访问到对象的属性 
  但是早期没有class，函数里面需要this来完成OOP 如构造器
- js 做了一个不好的设计
  this 并不是一直指向对象本身 而是由函数的调用方式决定的
  - 作为普通函数运行，this指向全局对象
    this 是没有必要的，只要沿着作用域链查找就可以了
    js函数特别灵活，js作者忘了处理这种情况
    但是this总要指向一个地方，js作者偷懒 指向了全局
    var 声明的变量，会挂载到全局window对象上
    不好的 容易造成全局变量window的污染 
    let 声明的变量，不会挂载到全局window对象上
    严格模式也规范了没有必要的this指向，直接报错
  - 作为对象的方法来调用，this指向此对象
  - js 执行机制根编译阶段挂钩
  - this 是个例外，由调用方式(执行阶段)决定
    this 是一个指针 指向调用它的对象
    谁调用这个方法，this就指向谁

## this 指向的各种情况
- 作为对象的方法被调用，指向对象
- 作为普通函数被调用，指向全局对象，严格模式undefined
- call,apply 绑定this
- 构造函数调用 this 指向实例
- 事件处理函数 this指向事件绑定的元素
