## 内存空间
- 代码空间
  代码从硬盘 读取到内存中 
- 栈内存
  调用栈(js执行的主角) 
  快，好管理，大小固定
  调用栈的栈顶指针切换要快；
  因为 空间 小，连续，可以切换频繁
- 堆内存
  打辅助
  大对象的空间，分配更耗时
  对象动态的，不连续的

## 总结
  JS v8引擎需要用栈俩维护程序执行上下文的状态。
  如果栈空间太大了，或不连续(复杂数据类型也放在栈中),那么就会影响上下文切换的效率，
  进而影响整个程序的执行效率
  执行上下文的切换，其实是执行栈栈顶指针的偏移。
  栈空间，用来存放原始数据类型，空间占据不大，且连续。
  堆内存很大，能存放很多很大的数据，不过缺点是分配内存和回收内存都会占用一定的时间

  - 简单数据类型 直接使用
  - 复杂数据类型 二传手
    回收的时候 栈先回收(指针偏移) 堆内存中的对象没有变量引用，慢慢回收。

## 内存机制理解闭包
  - 执行流程
    - 先编译 foo函数，创建一个空的执行上下文 (全局执行上下文)
    - 执行foo 之前，编译过程，对 setname，getname 快速地词法扫描
      闭包内部函数，myname test1 由内部函数引用的自由变量，JS判断有闭包，在堆空间创建一个closure(foo)
    - getname setname 执行时可以快速访问到closure(foo)

  闭包的核心 第一步 需要扫描内部函数 堆内存中
  第二步 把内部函数引用的外部变量保存到堆中
